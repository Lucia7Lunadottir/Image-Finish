## 1. Запуск

**Точка входа:** `main.py`

```
python main.py                  # запустить программу
python main.py /путь/к/файлу    # открыть файл при запуске
```

Что делает `main.py`:
1. Загружает шрифты из папки `fonts/` (файлы `.ttf`, `.otf`)
2. Создаёт главное окно `MainWindow`
3. Если передан аргумент — открывает файл

**Папки рядом с `main.py`:**
- `fonts/` — сюда кладите файлы шрифтов `.ttf`/`.otf`, они подхватятся автоматически
- `brushes/` — пресеты кистей
- `patterns/` — паттерны для штампа

## 3. Главная идея

Вся программа работает по такой цепочке:

```
Мышь/Клавиатура
       ↓
  CanvasWidget          (холст, ui/canvas_widget.py)
       ↓
  active_tool           (текущий инструмент, tools/*.py)
       ↓
  Document / Layer      (данные, core/document.py, core/layer.py)
       ↓
  get_composite()       (склейка всех слоёв в одно изображение)
       ↓
  paintEvent()          (отрисовка на экране)
```

**Три главных объекта:**

| Объект | Что хранит | Файл |
|--------|-----------|------|
| `Document` | Размер холста, список слоёв, выделение | `core/document.py` |
| `Layer` | Одно изображение (пиксели) + все настройки слоя | `core/layer.py` |
| `CanvasWidget` | Масштаб, смещение, текущий инструмент, история | `ui/canvas_widget.py` |

## 4. Document

**Файл:** `core/document.py`

Документ — это весь файл изображения. Он содержит:

```python
doc.width           # ширина холста (пиксели)
doc.height          # высота холста (пиксели)
doc.layers          # список слоёв [Layer, Layer, ...]
doc.active_layer_index  # индекс активного слоя (0 = первый снизу)
doc.selection       # текущее выделение (QPainterPath или None)
doc.color_mode      # "RGB" или "CMYK" и т.д.
doc.guides_v/h      # вертикальные/горизонтальные направляющие
doc.show_grid       # показывать сетку?
doc.work_path       # рабочий контур (для инструмента Перо)
```

### Методы Document

| Метод | Что делает |
|-------|-----------|
| `doc.add_layer(name, index, fill_color)` | Добавить новый слой |
| `doc.duplicate_layer(index)` | Дублировать слой |
| `doc.remove_layer(index)` | Удалить слой |
| `doc.move_layer(from_index, to_index)` | Переместить слой вверх/вниз |
| `doc.get_active_layer()` | Получить текущий слой |
| `doc.get_composite()` | Склеить все слои в одно изображение |
| `doc.flatten()` | Свести все слои в один |
| `doc.apply_crop(rect)` | Кадрировать документ |
| `doc.trim_transparent()` | Обрезать прозрачные края |
| `doc.reveal_all()` | Расширить холст до всех пикселей |
| `doc.snapshot_layers()` | Сделать снимок для истории |
| `doc.restore_layers(snapshot)` | Восстановить из снимка |
| `doc.save_to_imfn(path)` | Сохранить в формате `.imfn` |
| `doc.load_from_imfn(path)` | Загрузить из `.imfn` |

### Как склеиваются слои: `get_composite()`

Метод проходит по слоям снизу вверх и рисует каждый с учётом:
- видимости (`layer.visible`)
- прозрачности (`layer.opacity`)
- режима наложения (`layer.blend_mode`)
- маски слоя (`layer.mask`)
- векторной маски (`layer.vector_mask`)
- обтравочной маски (`layer.clipping`)
- корректирующего слоя (применяет коррекцию к уже нарисованным ниже пикселям)

## 5. Layer

**Файл:** `core/layer.py`

Слой — это один прямоугольник пикселей плюс куча настроек.

### Все поля Layer

```python
layer.name          # название ("Background", "Layer 1", ...)
layer.layer_id      # уникальный ID (UUID строка)
layer.parent_id     # ID родительской группы/артборда (или None)
layer.visible       # True/False — виден ли слой
layer.locked        # полная блокировка (нельзя ничего)
layer.lock_alpha    # нельзя менять прозрачность
layer.lock_pixels   # нельзя рисовать
layer.lock_position # нельзя двигать
layer.opacity       # прозрачность 0.0..1.0
layer.blend_mode    # режим наложения ("Normal", "Multiply", ...)
layer.offset        # смещение слоя QPoint(x, y)
layer.image         # само изображение (QImage, пиксели)
layer.mask          # маска слоя (QImage или None)
layer.mask_enabled  # включена ли маска
layer.editing_mask  # сейчас редактируем маску?
layer.vector_mask   # векторная маска (QPainterPath или None)
layer.clipping      # обтравочная маска (True = клипуется к слою ниже)
layer.layer_type    # тип: "raster" | "text" | "adjustment" | "fill" |
                    #       "smart_object" | "artboard" | "group" | "frame"
layer.text_data     # данные текстового слоя (dict или None)
layer.shape_data    # данные векторного слоя
layer.adjustment_data # данные корректирующего слоя
layer.fill_data     # данные слоя заливки
layer.layer_styles  # стили слоя (тени, обводки и т.д.)
layer.artboard_rect # прямоугольник артборда (QRect или None)
```

### Типы слоёв и что они означают

| `layer_type` | Описание | Где данные |
|-------------|---------|-----------|
| `"raster"` | Обычный пиксельный слой | `layer.image` |
| `"text"` | Текстовый слой | `layer.text_data` |
| `"adjustment"` | Корректирующий слой | `layer.adjustment_data` |
| `"fill"` | Слой заливки (цвет/градиент) | `layer.fill_data` |
| `"smart_object"` | Смарт-объект | `layer.smart_data` |
| `"group"` | Группа слоёв | дочерние слои имеют `parent_id` |
| `"artboard"` | Артборд | `layer.artboard_rect` |
| `"frame"` | Рамка | `layer.frame_data` |

### Режимы наложения (`blend_mode`)

Строки, которые понимает программа:
`"Normal"`, `"Multiply"`, `"Screen"`, `"Overlay"`,
`"Darken"`, `"Lighten"`, `"ColorDodge"`, `"ColorBurn"`,
`"HardLight"`, `"SoftLight"`, `"Difference"`, `"Exclusion"`

Соответствие строк и режимов Qt описано в `core/document.py`, функция `_get_composition_mode()`.

## 6. History

**Файл:** `core/history.py`

История хранит снимки состояния документа. Максимум 40 шагов (можно изменить в `HistoryManager(max_states=40)`).

### Как работает

```
Перед каждым действием:
  _push_history("Описание действия")
    → сохраняет snapshot слоёв в стек отмен

Ctrl+Z (отмена):
  undo() → берёт последний snapshot из стека, восстанавливает

Ctrl+Shift+Z (повтор):
  redo() → восстанавливает из стека повторов
```

### Структура `HistoryState`

```python
HistoryState(
    description="Brush stroke",   # название в панели истории
    layers_snapshot=...,          # снимок всех слоёв
    active_layer_index=0,         # какой слой был активен
    doc_width=800, doc_height=600,# размер документа
    selection_snapshot=...,       # снимок выделения
    work_path_snapshot=...,       # снимок контура
)
```

### Кто вызывает `_push_history`

Все действия, которые меняют изображение, вызывают `_push_history(описание)` **перед** изменением. Это делается в:
- `ui/actions/layer_actions.py` — действия со слоями
- `ui/actions/edit_actions.py` — вставка, заливка и т.д.
- `ui/canvas_widget.py` — при нажатии мышью (если инструмент вернул `needs_history_push() = True`)

## 7. Инструменты

**Базовый файл:** `tools/base_tool.py`

Каждый инструмент — это Python-класс, который наследуется от `BaseTool`. Инструмент получает события мыши и изменяет документ.

### Методы BaseTool

```python
class BaseTool:
    name = "Tool"        # внутреннее имя (используется как ключ в реестре)
    icon = "🔧"          # значок в тулбаре
    shortcut = ""        # буква горячей клавиши
    modifies_canvas_on_move = False  # нужно ли перерисовывать на каждый move?

    def on_press(self, pos, doc, fg, bg, opts):
        """Вызывается при нажатии кнопки мыши."""
        # pos — QPoint с координатами в пространстве документа
        # doc — объект Document
        # fg  — основной цвет (QColor)
        # bg  — фоновый цвет (QColor)
        # opts — словарь с опциями инструмента (размер кисти и т.д.)

    def on_move(self, pos, doc, fg, bg, opts):
        """Вызывается при движении мыши с зажатой кнопкой."""

    def on_release(self, pos, doc, fg, bg, opts):
        """Вызывается при отпускании кнопки мыши."""

    def cursor(self):
        """Возвращает курсор для этого инструмента."""
        return Qt.CursorShape.CrossCursor

    def needs_history_push(self):
        """Нужно ли сохранять снимок перед on_press?"""
        return True
```

### Координатная система

- `pos` в `on_press/on_move/on_release` — координаты **в пространстве документа** (не экрана)
- `(0, 0)` = левый верхний угол изображения
- `(doc.width-1, doc.height-1)` = правый нижний угол
- Масштаб и прокрутка уже учтены — инструмент всегда работает в пикселях изображения

### Словарь `opts` — что в нём есть

| Ключ | Тип | Описание |
|------|-----|---------|
| `"_ctrl"` | bool | зажат ли Ctrl |
| `"_alt"` | bool | зажат ли Alt |
| `"_shift"` | bool | зажат ли Shift |
| `"brush_size"` | int | размер кисти в пикселях |
| `"brush_opacity"` | float | непрозрачность 0.0–1.0 |
| `"brush_hardness"` | float | жёсткость кисти 0.0–1.0 |
| `"brush_angle"` | float | угол кисти в градусах |
| `"brush_mask"` | str | форма: `"round"`, `"square"`, `"scatter"` |
| `"brush_mirror_x"` | bool | зеркало по X |
| `"brush_mirror_y"` | bool | зеркало по Y |
| `"fill_tolerance"` | int | допуск заливки 0–255 |
| `"fill_sample_all"` | bool | заливка по всем слоям |
| `"move_auto_select"` | bool | автовыбор слоя при перемещении |

Опции устанавливаются из панели опций инструмента (сверху) через сигнал `option_changed`.

### Как инструмент рисует на холсте

Типичный паттерн (пример кисти):

```python
def on_press(self, pos, doc, fg, bg, opts):
    layer = doc.get_active_layer()
    if layer is None or layer.locked:
        return
    # Получить изображение слоя:
    img = layer.image
    # Нарисовать на нём с QPainter:
    p = QPainter(img)
    p.setBrush(fg)
    p.drawEllipse(pos, radius, radius)
    p.end()
    # НЕ нужно звать update() — canvas_widget делает это сам
```

### Специальные инструменты: "stroke preview"

Кисть и похожие инструменты используют механизм **stroke preview** (предпросмотр мазка) чтобы не модифицировать `layer.image` до окончания мазка:

1. В `on_press` — инициализируют `layer.active_stroke` (словарь с временным изображением)
2. В `on_move` — рисуют в `active_stroke["img"]`
3. В `on_release` — применяют `active_stroke` к `layer.image` и очищают

`active_stroke` — это dict:
```python
layer.active_stroke = {
    "img": QImage(...),           # временный холст для мазка
    "mode": CompositionMode_...,  # режим наложения
    "opacity": 0.5,               # непрозрачность мазка
    "clip": QPainterPath(...)     # обрезать по выделению (или None)
}
```

Создайте новый файл `tools/stamp_tool.py` или добавьте класс в существующий файл:

```python
from PyQt6.QtCore import Qt, QPoint
from PyQt6.QtGui import QPainter, QColor
from tools.base_tool import BaseTool


class StampTool(BaseTool):
    name = "Stamp"          # ← уникальное имя (используется как ключ)
    icon = "🖼️"             # ← значок (emoji или строка)
    shortcut = "K"          # ← горячая клавиша (необязательно)
    modifies_canvas_on_move = True  # True если нужно перерисовывать при движении

    def __init__(self):
        self._drawing = False   # отслеживаем: нажата ли кнопка

    def on_press(self, pos, doc, fg, bg, opts):
        self._drawing = True
        self._paint(pos, doc, fg, opts)

    def on_move(self, pos, doc, fg, bg, opts):
        if self._drawing:
            self._paint(pos, doc, fg, opts)

    def on_release(self, pos, doc, fg, bg, opts):
        self._drawing = False

    def _paint(self, pos, doc, fg, opts):
        layer = doc.get_active_layer()
        # Проверки: слой есть, не заблокированы пиксели
        if layer is None or layer.lock_pixels:
            return
        size = opts.get("brush_size", 20)
        p = QPainter(layer.image)
        p.setRenderHint(QPainter.RenderHint.Antialiasing)
        p.setBrush(fg)
        p.setPen(Qt.PenStyle.NoPen)
        p.drawRect(pos.x() - size//2, pos.y() - size//2, size, size)
        p.end()

    def cursor(self):
        return Qt.CursorShape.CrossCursor

    def needs_history_push(self):
        return True   # сохранять снимок перед каждым нажатием
```

## 10. Опции инструмента

**Файл:** `ui/tool_options_bar.py`

Панель опций сверху — это стек виджетов (`QStackedWidget`). При переключении инструмента показывается нужная страница.

Каждая страница — это класс, наследованный от `BaseOptions` (`ui/tool_options/base_options.py`).

### Как опции передаются инструменту

1. Пользователь меняет ползунок в панели опций
2. Панель опций испускает сигнал `option_changed(ключ, значение)`
3. `MainWindow` перехватывает его и сохраняет в `canvas.tool_opts[ключ] = значение`
4. При следующем событии мыши `canvas_widget` передаёт `tool_opts` как параметр `opts` в `on_press/on_move/on_release`

### Кто какую опцию использует

Кисть читает из `opts`:
- `"brush_size"` — размер
- `"brush_opacity"` — непрозрачность
- `"brush_hardness"` — жёсткость
- `"brush_angle"` — угол
- `"brush_mask"` — форма (`"round"`, `"square"`, `"scatter"`)
- `"brush_mirror_x"` / `"brush_mirror_y"` — симметрия

Заливка:
- `"fill_tolerance"` — допуск
- `"fill_sample_all"` — по всем слоям

## 13. Добавить коррекцию

**Пример:** коррекция "Сепия" (sepia).

### Шаг 1: Создать файл коррекции

Создайте `core/adjustments/sepia.py`:

```python
from PyQt6.QtGui import QImage
from core.locale import tr
from ui.adjustments_dialog import _to_argb32, _in_place_arr, _AdjustDialog
from core.adjustments._widgets import _FSliderRow


def apply_sepia(src: QImage, intensity: float) -> QImage:
    """intensity: 0.0–1.0"""
    import numpy as np
    img = _to_argb32(src).copy()
    arr = _in_place_arr(img)    # numpy массив shape=(H, W, 4) BGRA
    # Сепия: формула R=0.393R+0.769G+0.189B, G=0.349R+0.686G+0.168B, B=...
    b = arr[:, :, 0].astype(float)
    g = arr[:, :, 1].astype(float)
    r = arr[:, :, 2].astype(float)
    sr = np.clip(r*0.393 + g*0.769 + b*0.189, 0, 255)
    sg = np.clip(r*0.349 + g*0.686 + b*0.168, 0, 255)
    sb = np.clip(r*0.272 + g*0.534 + b*0.131, 0, 255)
    arr[:, :, 0] = (b * (1 - intensity) + sb * intensity).astype(np.uint8)
    arr[:, :, 1] = (g * (1 - intensity) + sg * intensity).astype(np.uint8)
    arr[:, :, 2] = (r * (1 - intensity) + sr * intensity).astype(np.uint8)
    return img.convertToFormat(QImage.Format.Format_ARGB32_Premultiplied)


class SepiaDialog(_AdjustDialog):
    def __init__(self, layer, canvas_refresh, parent=None):
        super().__init__("Сепия", layer, canvas_refresh, parent)
        self._intensity = _FSliderRow("Интенсивность", 0, 100, 80, 100.0, 0)
        self._add_row(self._intensity)
        self._seal()

    def _apply_preview(self):
        self._layer.image = apply_sepia(self._orig_argb32, self._intensity.value())
        self._canvas_refresh()
```

## 16. Canvas Widget

**Файл:** `ui/canvas_widget.py`

Это самый большой и важный файл UI. Он отвечает за:
- Отрисовку документа на экране (`paintEvent`)
- Преобразование координат экран ↔ документ
- Масштабирование и прокрутку
- Вращение холста
- Передачу событий мыши в активный инструмент
- Отображение выделения (marching ants)
- Отображение курсора

### Ключевые поля CanvasWidget

```python
self.document       # Document
self.active_tool    # текущий инструмент
self.tool_opts      # dict с настройками инструмента
self.zoom           # масштаб (1.0 = 100%)
self._pan           # смещение холста (QPointF)
self._view_rotation # угол поворота вида (градусы)
self.history        # HistoryManager
self.filepath       # путь к открытому файлу
```

### Преобразование координат

```python
# Экранные → документные (учитывает zoom, pan, rotation)
doc_pos = canvas.to_doc(screen_pos)

# Документные → экранные
screen_pos = canvas.to_screen(doc_pos)
```

### Когда перерисовывается холст

Автоматически перерисовывается при вызове `canvas.update()`. Это происходит:
- Когда инструмент изменил документ (`on_move` вызывается только если `modifies_canvas_on_move = True`)
- После любого действия через `_canvas_refresh()` в MainWindow
- При изменении масштаба, прокрутки

### Отрисовка (порядок слоёв в `paintEvent`)

1. Шахматный узор (фон для прозрачности)
2. Составное изображение документа (`get_composite()`)
3. Сетка, линейки, направляющие
4. Выделение (marching ants)
5. Предпросмотр subtract-drag
6. Предпросмотр лассо
7. Курсор кисти (круг/квадрат)
8. Специальные превью инструментов (трансформация, рамка и т.д.)

### Специальные константы

```python
# ui/canvas_widget.py, начало файла:
_BRUSH_TOOLS = {"Brush", "Eraser", ...}    # инструменты с круглым курсором
_SELECTION_TOOLS = {"Select", "Lasso", ...} # инструменты выделения (курсор +/-)
```

### Как работает `.imfn`

Это gzip-сжатый pickle (бинарный формат Python). Формат данных:

```python
{
    "width": 800,
    "height": 600,
    "color_mode": "RGB",
    "bit_depth": 8,
    "layers": [layer.to_dict(), layer.to_dict(), ...]
}
```

Каждый слой сериализуется методом `layer.to_dict()` (изображения сохраняются как PNG-байты).

> **Важно:** если вы добавляете новое поле в `Layer`, нужно обновить `to_dict()` и `from_dict()` чтобы оно сохранялось/загружалось корректно.

## 22. Стили слоя (Layer Styles)

**Файл:** `core/layer_styles.py`

Стили слоя хранятся в `layer.layer_styles` — это словарь. Каждый стиль — это тоже словарь с ключом `"enabled": True/False` и параметрами.

Применяются в `get_composite()` через `apply_layer_styles(img, layer.layer_styles)`.

### Поддерживаемые стили

| Ключ | Название | Параметры |
|------|---------|---------|
| `"drop_shadow"` | Тень | `distance`, `angle`, `size`, `color`, `opacity`, `blend_mode` |
| `"outer_glow"` | Внешнее свечение | `size`, `color`, `opacity`, `blend_mode` |
| `"inner_shadow"` | Внутренняя тень | `distance`, `angle`, `size`, `color`, `opacity`, `blend_mode` |
| `"inner_glow"` | Внутреннее свечение | `size`, `color`, `opacity`, `blend_mode` |
| `"stroke"` | Обводка | `size`, `color`, `opacity`, `blend_mode` |
| `"bevel"` | Тиснение | `distance`, `angle`, `size`, `color`, `shadow_color`, `opacity` |
| `"satin"` | Глянец | `distance`, `angle`, `size`, `color`, `opacity`, `blend_mode` |
| `"color_overlay"` | Наложение цвета | `color`, `opacity`, `blend_mode` |
| `"gradient_overlay"` | Наложение градиента | `stops`, `type`, `angle`, `opacity` |
| `"pattern_overlay"` | Наложение паттерна | `pattern` (путь к файлу), `scale`, `opacity`, `blend_mode` |
| `"offset"` | Смещение | `dx_pct`, `dy_pct`, `edge_mode` (`"wrap"`, `"repeat"`, `"transparent"`) |

### Пример

```python
layer.layer_styles = {
    "drop_shadow": {
        "enabled": True,
        "distance": 10,
        "angle": 90,
        "size": 8,
        "color": QColor(0, 0, 0),
        "opacity": 70,
        "blend_mode": "Multiply"
    },
    "stroke": {
        "enabled": True,
        "size": 3,
        "color": QColor(255, 0, 0),
        "opacity": 100,
        "blend_mode": "SourceOver"
    }
}
```

## 23. Утилиты для работы с пикселями

**Файл:** `tools/tool_utils.py`

Вспомогательные функции, которые используют numpy и QImage. Импортируются во многих инструментах.

### Преобразование QImage ↔ numpy

```python
from tools.tool_utils import _qimage_to_np, _np_to_qimage

arr = _qimage_to_np(img)   # QImage → numpy (H, W, 4) BGRA uint8
img = _np_to_qimage(arr)   # numpy → QImage
```

> **Порядок каналов в numpy:** `arr[:, :, 0]` = B (синий), `arr[:, :, 1]` = G (зелёный), `arr[:, :, 2]` = R (красный), `arr[:, :, 3]` = A (прозрачность)

### Маски кисти

```python
from tools.tool_utils import _brush_mask, _circle_mask, _soft_circle_mask

# Маска с контролем жёсткости
mask = _brush_mask(rect, cx, cy, radius, hardness=0.8)
# → numpy (H, W, 1) float32, значения 0.0–1.0

# Бинарная круглая маска
mask = _circle_mask(rect, cx, cy, radius)

# Мягкая маска (smoothstep от центра)
mask = _soft_circle_mask(rect, cx, cy, radius)
```

### Размытие

```python
from tools.tool_utils import _box_blur_rgb, _box_blur_np, _sharpen_np

# Box blur для float32 RGB (H, W, 3)
blurred = _box_blur_rgb(arr_float, radius=5)

# Box blur для uint8 BGRA (H, W, 4) — с учётом premultiplied alpha
blurred = _box_blur_np(arr_uint8, radius=3)

# Резкость через unsharp mask
sharpened = _sharpen_np(arr_uint8, strength=1.5)
```

### Вспомогательная функция `_clamp_rect`

```python
from tools.tool_utils import _clamp_rect

# Обрезает QRect по границам изображения
safe_rect = _clamp_rect(some_rect, image_width, image_height)
```

## 24. Система переводов (Locale)

**Файл:** `core/locale.py`

### Как использовать

```python
from core.locale import tr

text = tr("menu.file")           # получить перевод
text = tr("status.zoom", pct=75) # с подстановкой: "Масштаб: 75%"
```

### Как добавить новый язык

1. Создайте файл `locales/xx.json` (где `xx` — код языка, например `de`, `fr`, `uk`)
2. Файл — это JSON-объект, пример:

```json
{
  "_name": "Deutsch",
  "menu.file": "Datei",
  "menu.edit": "Bearbeiten",
  "tool.brush": "Pinsel"
}
```

3. Программа автоматически найдёт файл и добавит язык в меню при следующем запуске

### Где хранятся файлы переводов

```
image-finish/
└── locales/
    ├── en.json   ← английский
    ├── ru.json   ← русский
    └── ...       ← ваши новые языки
```

### Текущий язык сохраняется в `settings.json`

```json
{
  "language": "ru"
}
```

При запуске программа автоматически загружает последний выбранный язык.
